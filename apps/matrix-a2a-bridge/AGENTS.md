- `internal/config/` â€” typed, env-parsed config (`caarlos0/env`); `Config`, `Load` (fail-fast validation, incl. timeouts/rates).
- `internal/matrixapp/` â€” builds the mautrix `AppService` (`CreateFull`, optional SQL StateStore), loads/generates the registration (`rate_limited: false`), and wraps the transaction route with exact-body, bounded, authenticated pre-ACK durable intake.

## Conventions

- Server name `fgentic.fmind.ai`; bot `@a2a-bridge`; ghosts `@agent-<name>` (exclusive appservice namespace). Agent A2A path: `/api/a2a/<namespace>/<name>`.
- Remote mappings are fail-closed trust pins, not URL shortcuts. Require a positive timeout and token budget, matching name/provider organization/protected key ID, an ES256 signature that verifies under the pinned P-256 public JWK, a JSONRPC or HTTP+JSON interface at the configured URL, and the `https://fgentic.fmind.ai/a2a/extensions/token-budget/v1` card extension. The timeout is an additional whole-delegation ceiling combined with global request/task deadlines; start the whole-task clock at the first persisted A2A attempt boundary, never at ledger admission, so room backlog does not consume it. The token budget is partner-enforced extension metadata, not locally observed model-token accounting. An unsigned, mismatched, or tampered refresh makes the target unready before limiter or A2A admission and audits `terminal_stage=agent_card`, `terminal_reason=agent_card_untrusted`, and `a2a_attempted=false`.
- The appservice is a strict single ready intake consumer: Helm rejects `replicaCount != 1`, and Deployment updates use a zero-surge rollout (`maxSurge: 0`, `maxUnavailable: 1`). Postgres leases recover work across replacement processes; this does not make HTTP intake highly available.
- Rooms are **unencrypted** by design (the crypto package is not wired) â€” see repo ADR 0008.
- Treat `allowedRooms` as an exact authorization boundary and require current ghost membership before every fresh or resumed A2A action. Only `ACCESS_MANAGER_MXID` may invite a mapped ghost, and only into its declared room binding. Never add `EnsureJoined` to message, refusal, recovery, directory, budget, or failure-notice ghost paths; a Matrix message cannot create authority. Managed rooms are private, invite-only, room v12+, with the access manager responsible for invite/kick/ban and authorization state.
- A2A is **non-streaming** (`SendMessage`); route through agentgateway by default (`A2A_BASE_URL`).
- Durable intake must commit the exact transaction-body SHA and every eligible per-target job in one database transaction before HTTP 200. Exact replays are idempotent; a changed body under the same transaction ID is HTTP 409. Keep `APPSERVICE_TRANSACTION_MAX_BYTES` positive and bounded.
- `ROOM_QUEUE_CAPACITY` and `GLOBAL_QUEUE_CAPACITY` bound all non-terminal durable jobs, including delayed and leased work. Serialize count-and-insert across transactions; check room capacity before global capacity for each new target and count accepted targets earlier in the same batch. A refusal must commit a content-scrubbed terminal `denied` row with the stable `queue_room_capacity_rejected` or `queue_global_capacity_rejected` reason. For a newly accepted transaction, consume `AdmissionResult.CapacityDenied` once and emit the existing single `queue_full` audit/metric contract (`terminal_stage=queue`, `rate_limit_verdict=not_checked`, `a2a_attempted=false`) without A2A or rate admission. After transaction consumption, hand only content-free correlation fields to a fixed-size best-effort notice worker; it must use the shared notice buckets and may suppress at either bounded handoff. Exact replay must not emit the audit, metric, or notice again.
- `CONCURRENCY` bounds active leased jobs. The oldest non-terminal job blocks later work only in its own room; lease owner plus monotonically increasing generation fences every mutation. Heartbeat before half the lease duration, and release failed work with bounded backoff instead of polling Postgres from one goroutine per idle worker.
- Persist the deterministic A2A message ID and content-free attempt boundary in `a2a_prepared` before invoking the client, but never infer target receipt or idempotency from them. A pre-transport failure may retry; once transport may have started, transition to `ambiguous`, post a generic notice, and never resend. A persisted task ID resumes only through `GetTask`.
- Matrix reply, placeholder, edit, and per-index artifact-event stages use distinct deterministic transaction IDs. Persist the result/notice in `reply_pending` before projection, record the primary returned event ID immutably, and commit the conversation context in the same database transaction as its job transition. Media uploads can repeat after a crash; event creation must remain idempotent even when the content repository keeps an unreferenced duplicate upload.
- Retry limits count consecutive recovery failures; successful transitions and healthy polls reset the count. Exhaustion becomes `dead`. Terminal transitions scrub prompt/event/result content immediately; retain non-content `delivered`/`denied` tombstones for at least 24 hours and `ambiguous`/`dead` evidence indefinitely. Preserve legacy processed-event markers for their minimum 24-hour compatibility window.
- Each of the four invocation/notice sender/room limiter maps is independently capped at 4096 buckets. Unknown keys fail closed at capacity; never evict an active bucket to admit churn because that resets its burst budget. Idle cleanup scans at most once per minute and only on a new key.
- Shutdown must stop bridge-owned HTTP intake, force-close timed-out transaction connections before the synchronous processor barrier, stop new durable claims, and drain active leases for the configured 25-second grace under the chart's 45-second pod grace. Cancellation after the grace must leave non-terminal jobs recoverable after lease expiry; never force a fake terminal state just to make shutdown green.
- Interactive durable work uses the bounded `bridge_delegation_controls` outbox under the parent job's fenced lease. Persist authorized cancel/continuation intent before acknowledgement; never resend a prepared A2A cancel or continuation after recovery. Replay question/progress projections only with their deterministic Matrix transaction IDs, and reconcile pin/unpin against current room state. Typing alone remains process-local and is not recovery evidence.
- Unit and contract tests cover persisted-boundary recovery, including lost-ACK ambiguity, known-task polling, bounded controls, and Matrix transaction replay. `mise run test:availability` covers graceful SIGTERM; `mise run test:crash-recovery` SIGKILLs the real bridge at twelve persisted job/control boundaries. Run the latter on the candidate revision before claiming those workflows, and never promote either fixture into node-loss evidence or a production RTO.
- The full projected agent ConfigMap directory must be mounted, never `agents.yaml` via `subPath`: the latter cannot receive Kubernetes atomic updates. Invalid reloads keep the last-known routing policy.
- Agent routing files declare `schemaVersion: 1`; an unknown major fails closed at startup/reload. Missing versions remain a deprecated v1 compatibility path only, and every additive or breaking schema change must follow `docs/stability.md`.
- Remote cards refresh independently every five minutes in production. The provider-free kind fixture uses one second and a fixed public test-only P-256 identity; its valid round trip must assert the message metadata and `A2A-Extensions` token-budget contract. It then mutates the card after signing and must prove zero further remote dispatches plus exactly one bounded untrusted-card audit.
- Matrix display names and configured `mxc://` avatars are portable. Arbitrary description fields require Matrix v1.16 and are not consistently rendered by Element, so keep `!agents` as the user-facing metadata/status surface.
- Errors as values, wrapped with `%w`; never ignore an `err`. Context first for I/O, with a deadline on every A2A call. Never pass provider-controlled A2A error text to `span.RecordError`; represent trace failures with fixed-name, attribute-free events plus bounded outcome, terminal-stage, terminal-reason, and status fields. `log/slog` (JSON); stable audit records use the dedicated `log_stream=audit` child logger and must never include Matrix/A2A content bodies.
- Definition of done: `mise run format` clean, `mise run check` no findings, `mise run test` green, new behavior covered by a test. Conventional Commits; no attribution.
